# Security Policy

## Supported versions
This repository is documentation-heavy. Security issues typically relate to workflow configuration, tokens, or supply-chain risk.

| Component | Supported |
|---|---|
| GitHub Actions workflows | ✅ Yes |
| Validation scripts | ✅ Yes |

## Reporting a vulnerability
Please report security issues privately.

- Email: security@tech4life-beyond.com (preferred)
- If email is unavailable, open a GitHub Security Advisory (Private)

## What to include
- Steps to reproduce
- Impact assessment
- Proposed mitigation (if any)
- Any relevant logs or screenshots

## Disclosure policy
We aim to acknowledge receipt within 3 business days and provide an initial assessment within 10 business days.
